# print_Multi_Format パフォーマンス改善計画

## 問題の分析

現在のコードには以下のパフォーマンスボトルネックが存在します：

1. **COMオブジェクトの起動/終了コスト**
   - 各ファイルごとにExcel/Wordアプリケーションを起動・終了
   - COM初期化は重い処理（数秒かかる場合あり）

2. **プリンタ設定の繰り返し適用**
   - 各ファイルでプリンタハンドルを開閉
   - 同じプリンタに対して同じ設定を繰り返し適用

3. **シリアル処理**
   - ファイルを一つずつ順次処理
   - 大量ファイルの場合、合計時間が線形的に増加

4. **固定遅延待機**
   - PDF印刷後に固定5秒待機（PRINTER_JOB_WAIT_SECONDS）
   - 印刷ジョブの完了を監視せず、過剰な待機または不足

5. **エラーハンドリングの不備**
   - 例外処理が不完全でCOMオブジェクトが残存
   - 印刷されないファイルがあるとの報告

6. **ログ出力のオーバーヘッド**
   - 各ステップでファイルIOが発生

## 改善目標

- 単一ファイルの印刷開始までの待ち時間を50%削減
- 大量ファイル（10個以上）の処理時間を70%削減
- 印刷中の応答なし状態を解消
- 印刷されないファイルの問題を解決

## 具体的な改善策

### 1. COMオブジェクトの再利用（シングルトンパターン）
```python
class COMApplicationManager:
    _excel_app = None
    _word_app = None
    
    @classmethod
    def get_excel_app(cls):
        if cls._excel_app is None:
            cls._excel_app = win32com.client.Dispatch("Excel.Application")
            cls._excel_app.Visible = False
        return cls._excel_app
    
    @classmethod
    def cleanup(cls):
        # アプリケーションを終了（必要に応じて）
        pass
```

### 2. 非同期処理による並列化
- `concurrent.futures.ThreadPoolExecutor`を使用
- ファイル形式ごとにスレッドプールを分離（COMオブジェクトのスレッド安全性考慮）
- 最大同時実行数を制限（例：Excel: 2, Word: 2, PDF: 4）

### 3. プリンタ設定のキャッシュ
```python
_printer_cache = {}

def get_printer_settings(printer_name, orientation):
    cache_key = (printer_name, orientation)
    if cache_key not in _printer_cache:
        # 設定適用
        _printer_cache[cache_key] = True
    return True
```

### 4. 印刷ジョブの状態監視
- `win32print.EnumJobs()`で印刷ジョブの状態を監視
- ポーリング間隔を動的に調整（例：0.5秒間隔）
- タイムアウト設定（例：最大60秒）

### 5. エラーハンドリングの強化
- 各印刷操作の例外を詳細にキャッチ
- COMオブジェクトのクリーンアップを確実に実行
- リトライメカニズムの導入（例：最大2回リトライ）

### 6. プログレス表示の追加
- コンソールに進捗バーを表示
- 残り時間の推定表示
- 詳細ログはオプション化

### 7. 設定ファイルの導入
- 最大同時実行数
- タイムアウト設定
- リトライ回数
- ログレベル

## リスクと対応策

1. **COMオブジェクトのスレッド安全性**
   - 対応: 形式ごとに専用スレッドプールを使用
   - 対応: ロックによる排他制御

2. **プリンタの同時印刷制限**
   - 対応: 同時実行数を制限（デフォルト: Excel 2, Word 2, PDF 4）

3. **メモリリーク**
   - 対応: 定期的なCOMオブジェクトのクリーンアップ
   - 対応: メモリ使用量の監視
